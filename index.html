Documentation with direction

Listed here are several of my best Portfolio examples that I have decided to include, ranging from software manuals to comprehensive how-to tutorials tailored for specialized audiences. Each example has a description attached that lists the challenges, purpose, and intended audience.

Git Tutorial for Technical Writers

An introductory Git tutorial for technical writers using Windows. The purpose of this document is to help guide technical writers seeking to learn how to contribute to documentation in version control developer environments.

Since the guide is targeted towards technical writers having difficulty understanding Git, the tone is casual in order to reduce the potentially confusing formality of syntax and procedure.

The challenges included formatting the documentation in a coherent structure that would make the abstract features of Git easy to understand. In this case, the solution was a glossary of commands and definitions, as well as a section devoted to elucidating any terminology.

Excel Trend Analyzer Tool Manual

A manual documenting an Excel program coded in VS Basic. The purpose of this document is to document the functions and operating procedure for this particular program. It lists the features of the program and describes how to use them.

The client designed the program for usage in an internal work environment. Thus, it could be concluded that the target audience already knows the context and any relevant information regarding data retrieval and application.For this reason, the tone utilized is formal, unambiguous, and concise.

Many of the challenges stemmed from having to communicate with the developer and having to translate rudimentary documentation/code from Polish to English. All of the communication was in Polish. Having a copy of the source code and personally using the program, as well as parsing through the code, helped shape the documentation.
